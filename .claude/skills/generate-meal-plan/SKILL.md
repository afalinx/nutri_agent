# Генерация плана питания

## Описание
Этот скилл генерирует персонализированный план питания на 1 день для пользователя. Агент выступает как AI-нутрициолог: получает контекст (профиль + рецепты из БД), составляет план в строгом JSON-формате и валидирует результат.

## Когда использовать
- Пользователь просит составить рацион / план питания / меню на день
- Пользователь говорит "сгенерируй план", "составь рацион", "что мне есть"

## Инструкции

### Шаг 1: Узнать пользователя
Если user_id не указан, выполни:
```bash
cd backend && PYTHONPATH=. python cli.py users
```
Покажи список и спроси, для кого генерировать.

### Шаг 2: Получить контекст
```bash
cd backend && PYTHONPATH=. python cli.py context --user-id <USER_ID> --day <DAY_NUMBER>
```
Это вернёт JSON с:
- `user` — профиль (цель, калораж, аллергии)
- `available_recipes` — рецепты из БД (id, title, КБЖУ, ингредиенты)
- `output_schema` — схема ответа

### Шаг 3: Составить план
На основе контекста составь JSON плана питания. **Строгие правила:**

1. **ТОЛЬКО рецепты из `available_recipes`**. Используй точные `id` из списка.
2. **Суммарные калории за день** должны быть в пределах ±5% от `user.target_calories`.
3. **Обязательные приёмы пищи:** breakfast, lunch, dinner. Snack опционален.
4. **Распределение:** завтрак ~25%, обед ~35%, ужин ~30%, перекус ~10%.
5. **Аллергии учтены** — рецепты уже отфильтрованы, но перепроверь ингредиенты.
6. **КБЖУ каждого блюда** бери из `available_recipes`, не выдумывай.

Формат ответа:
```json
{
  "daily_target_calories": <int>,
  "day": {
    "day_number": <int>,
    "total_calories": <float>,
    "total_protein": <float>,
    "total_fat": <float>,
    "total_carbs": <float>,
    "meals": [
      {
        "type": "breakfast",
        "recipe_id": "<uuid из available_recipes>",
        "title": "<название>",
        "calories": <float>,
        "protein": <float>,
        "fat": <float>,
        "carbs": <float>,
        "ingredients_summary": [
          {"name": "<ингредиент>", "amount": <float>, "unit": "<g/ml/шт>"}
        ]
      }
    ]
  }
}
```

### Шаг 4: Валидация
Сохрани JSON в файл и провалидируй:
```bash
cd backend && PYTHONPATH=. python cli.py validate --file /tmp/plan.json
```
Если `"valid": false` — исправь план (обычно нужно заменить блюдо, чтобы попасть в калораж) и повтори валидацию.

### Шаг 5: Сохранение (опционально)
Если пользователь хочет сохранить план в БД:
```bash
cd backend && PYTHONPATH=. python cli.py save --user-id <USER_ID> --file /tmp/plan.json
```

## Пример
Для пользователя с целевым калоражом 2400 ккал, цель "maintain", без аллергий:
- Завтрак: Овсяная каша с ягодами (350 ккал) + Яичница с томатами (320 ккал) = ~670 ккал (28%)
- Обед: Куриная грудка с рисом (520 ккал) + Греческий салат (280 ккал) = ~800 ккал (33%)
- Ужин: Лосось на гриле с овощами (450 ккал) = ~450 ккал (19%)
- Перекус: Йогурт с гранолой (280 ккал) + Кефир с отрубями (140 ккал) = ~420 ккал (17%)
- **Итого: ~2340 ккал** (отклонение 2.5% — OK)
